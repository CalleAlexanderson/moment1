Arbetsprocess

Jag har använt mig av npm och parcel för denna uppgift. Jag började med att installera npm till katalogen med "npm init -y" för att få ett blankt projekt, sedan isntallerade jag parcel med "npm install parcel --save-dev".

I filen package.json som generades skrev jag in de kommande som visades i filmen och bekantade mig med funktionen av dessa, "start": "parcel", "build": "parcel build" som jag förstått är dessa inbyggda kommand som jag sedan har gett en egen kallelse metod för i detta fall "npm run start" för start och "npm run build" för build.

Efter jag gjorde detta borde jag ha gjort en first commit till git men jag var för insatt i att skapa menyn för navigation så jag glömde att göra det tills jag var en bit in i det arbetet. Jag skapade en .gitignore fil som ignorerade node_modules katalogen, parcel cache katalogen och dist katalogen då de skapas igen på den dator som repot kopieras till.

Efter min första commit gjorde jag klart navigationen och började läsa igen parcel docs för att hitta ett sätt att uppfylla detta krav i uppgiften: "Bakåtkompabilitet i genererade filer, så att exempelvis JavaScript transpileras så att dessa fungerar i så många webbläsare som möjligt." Det jag hittade var "browserlist" och detta läste jag om och testade olika värden för att se vilket som gav högst procent inkluderade webbläsaren och det värde jag kom fram till var "<0%" vilket inte uppnådde 100% (uppnådde 98.6%, testade här Browserslist ) men jag hittade inget självklart sätt att få det högre.

Efter jag uppfyllt detta gick jag tillbaka till docs för att hitta ett sätt att omvandla bilder, jag hittade ett sätt att använda js och ett sätt att använda html och jag tyckte html var mycket lättare så jag använde den samt skapade en sharp.config.json fil för att ge en kvalite på alla bilder utan att behöva skriva på varenda en. Jag satte kvaliten genom att lägga in alla bilderna och sedan ändra tills jag var tillräckligt nöjd med storlek och utseende, resultatet blev en kvalite på 38 på bilderna. På alla bilder satte jag också att de skulle konverteras till AVIF och WebP och sedan gav jag de en width på 300px.

Jag använde netlify för publicering där jag använde npm run build som build command och dist som publish directory. Jag hade en error när jag använda build kommandet så jag sökte upp error meddelandet på webben och hittade att "main": inte skulle vara med i package.json filen i parcel 2 så jag tog bort den.
